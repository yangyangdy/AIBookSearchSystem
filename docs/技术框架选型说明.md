# 技术框架选型说明

## 一、核心框架选择

### 1.1 Web框架：FastAPI

**选择理由：**
- ✅ 高性能：基于Starlette和Pydantic，性能接近NodeJS和Go
- ✅ 自动API文档：自动生成Swagger/OpenAPI文档
- ✅ 类型安全：基于Pydantic，支持类型检查和验证
- ✅ 异步支持：原生支持async/await，适合IO密集型任务
- ✅ 现代Python特性：支持Python 3.8+，使用现代Python语法

**适用场景：**
- 检索服务API（图片搜索接口）
- 健康检查接口
- 管理接口（进度查询、配置等）

### 1.2 数据库ORM：SQLAlchemy

**选择理由：**
- ✅ 成熟稳定：Python最流行的ORM框架
- ✅ 支持异步：SQLAlchemy 2.0支持async/await
- ✅ 灵活性强：支持原生SQL和ORM两种方式
- ✅ 连接池管理：自动管理数据库连接池

**适用场景：**
- MySQL数据读取（book_info_table）
- 可选：OCR文字存储到MySQL（如果采用混合存储方案）

### 1.3 异步HTTP客户端：aiohttp / httpx

**选择理由：**
- ✅ 异步IO：提高API调用并发性能
- ✅ 支持连接池：复用连接，提升性能
- ✅ 超时控制：支持请求超时设置
- ✅ 重试机制：可以配合tenacity实现自动重试

**选择：httpx（推荐）**
- 更现代的API设计
- 更好的类型提示
- 支持HTTP/2

**适用场景：**
- 调用阿里云Embedding API
- 调用阿里云OCR API
- 下载图片

### 1.4 配置管理：Pydantic Settings

**选择理由：**
- ✅ 类型安全：基于Pydantic，支持类型验证
- ✅ 环境变量支持：自动从环境变量读取配置
- ✅ 配置文件支持：支持YAML/JSON配置文件
- ✅ 验证机制：自动验证配置项的有效性

**适用场景：**
- MySQL连接配置
- Milvus连接配置
- 阿里云API配置
- 处理参数配置（批次大小、并发数等）

### 1.5 日志框架：loguru

**选择理由：**
- ✅ 简单易用：比标准logging更友好
- ✅ 自动格式化：自动处理日志格式
- ✅ 文件轮转：支持日志文件自动轮转
- ✅ 彩色输出：控制台彩色日志，便于调试

**适用场景：**
- 批量处理日志
- API请求日志
- 错误日志
- 性能日志

### 1.6 任务处理：concurrent.futures / asyncio

**选择理由：**
- ✅ 标准库：无需额外依赖
- ✅ 并发控制：ThreadPoolExecutor控制线程数
- ✅ 异步支持：asyncio支持异步并发
- ✅ 简单易用：API简单，易于理解

**适用场景：**
- 批量处理中的并发处理
- 向量化和OCR的并行调用

**注意：** 如果后续需要分布式任务队列，可以考虑Celery，但快速项目暂不需要。

### 1.7 图像处理：Pillow (PIL)

**选择理由：**
- ✅ 标准库：Python图像处理标准库
- ✅ 功能完整：支持各种图像操作
- ✅ 性能良好：C扩展，性能好

**适用场景：**
- 图片格式验证
- 图片可读性检查
- 感知哈希计算（配合imagehash库）

### 1.8 向量数据库客户端：pymilvus

**选择理由：**
- ✅ 官方客户端：Milvus官方Python客户端
- ✅ 功能完整：支持所有Milvus功能
- ✅ 类型提示：支持类型提示

**适用场景：**
- Milvus连接管理
- Collection创建和管理
- 向量插入和检索

### 1.9 文本处理：jieba

**选择理由：**
- ✅ 中文分词：专门针对中文分词
- ✅ 简单易用：API简单
- ✅ 性能良好：分词速度快

**适用场景：**
- OCR文字的关键词提取
- 文本分词处理

## 四、框架使用说明

### 4.1 FastAPI使用

**优势：**
- 自动生成API文档（/docs）
- 类型验证和自动序列化
- 异步支持，性能好

**示例：**
```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI(title="书籍封面向量检索系统")

class SearchRequest(BaseModel):
    image_url: str
    top_k: int = 10

@app.post("/api/v1/search")
async def search(request: SearchRequest):
    # 处理逻辑
    return {"results": [...]}
```

### 4.2 异步处理

**批量处理使用异步：**
- 使用`asyncio`和`aiohttp/httpx`进行异步API调用
- 使用`asyncio.gather`并行处理多个任务
- 提高IO密集型任务的性能

**示例：**
```python
import asyncio
import httpx

async def process_batch(records):
    async with httpx.AsyncClient() as client:
        tasks = [process_single(client, record) for record in records]
        results = await asyncio.gather(*tasks)
    return results
```

### 4.3 配置管理

**使用Pydantic Settings：**
```python
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    mysql_host: str
    mysql_port: int = 3306
    mysql_user: str
    mysql_password: str
    mysql_database: str
    
    class Config:
        env_file = ".env"
        env_file_encoding = "utf-8"

settings = Settings()
```

## 五、总结

### 5.1 核心框架

| 模块 | 框架 | 版本要求 |
|------|------|---------|
| Web框架 | FastAPI | >=0.104.0 |
| 数据库ORM | SQLAlchemy | >=2.0.0 |
| HTTP客户端 | httpx | >=0.25.0 |
| 配置管理 | Pydantic Settings | >=2.1.0 |
| 日志 | loguru | >=0.7.0 |
| 向量数据库 | pymilvus | >=2.3.0 |
| 图像处理 | Pillow | >=10.1.0 |
| 文本处理 | jieba | >=0.42.1 |

### 5.2 设计原则

1. **类型安全**：使用Pydantic进行类型验证
2. **异步优先**：IO密集型操作使用异步
3. **配置外部化**：配置通过环境变量和配置文件管理
4. **日志完善**：使用loguru记录详细日志
5. **错误处理**：完善的错误处理和重试机制

### 5.3 性能考虑

- FastAPI异步处理，支持高并发
- 使用连接池复用连接
- 批量处理使用并发，提高效率
- 合理的超时和重试机制
